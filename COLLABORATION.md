# 友達と一緒に編集する方法

## 重要なステップ：コラボレーターを追加する

友達がリポジトリを編集できるようにするには、**GitHubでコラボレーターとして追加する必要があります**。

### コラボレーターを追加する手順（リポジトリのオーナーが行う）

1. GitHubでリポジトリを開く: `https://github.com/KDGTA1/AI_blog`
2. リポジトリページの上部メニューから **Settings** をクリック
3. 左側のメニューから **Collaborators** をクリック
4. **Add people** ボタンをクリック
5. 友達のGitHubユーザー名またはメールアドレスを入力
6. 友達に招待メールが送信される
7. 友達が招待を承認すると、編集できるようになります

---

## 友達がやること（初回セットアップ）

### 1. 招待を承認する

- GitHubから送られてきた招待メールを開く
- または、GitHubにログインして通知を確認
- 招待を承認する

### 2. リポジトリをクローンする

```bash
git clone https://github.com/KDGTA1/AI_blog.git
cd AI_blog
```

### 3. セットアップを完了する

詳細は `SETUP.md` を参照してください。

## 編集のワークフロー

### 毎回編集を始める前に

```bash
# 1. 最新の変更を取得
git pull origin main
```

### 編集後

```bash
# 1. 変更をステージング
git add .

# 2. コミット（変更内容を記録）
git commit -m "変更内容の説明（例: 記事を追加）"

# 3. プッシュ（GitHubにアップロード）
git push origin main
```

## よくある問題と解決方法

### ❌ 「Permission denied」エラーが出る

**原因**: コラボレーターとして追加されていない、または招待を承認していない

**解決方法**:
1. リポジトリのオーナーに連絡して、コラボレーターとして追加してもらう
2. GitHubの通知を確認して、招待を承認する

### ❌ 「Repository not found」エラーが出る

**原因**: リポジトリへのアクセス権限がない

**解決方法**:
1. コラボレーターとして追加されているか確認
2. GitHubに正しいアカウントでログインしているか確認

### ❌ プッシュできない（「remote: Permission to ... denied」）

**原因**: SSHキーや認証情報が正しく設定されていない

**解決方法**:
1. HTTPSでクローンし直す: `git clone https://github.com/KDGTA1/AI_blog.git`
2. または、GitHubのPersonal Access Tokenを設定する

### ❌ 競合（コンフリクト）が発生した

**原因**: 同じファイルを複数人が同時に編集した

**解決方法**:
1. 最新の変更を取得: `git pull origin main`
2. 競合しているファイルを開いて、手動で修正
3. 修正後、再度コミット・プッシュ

---

## 安全に編集するためのコツ

1. **編集前に必ず `git pull` する** - 最新の状態を取得
2. **小さな変更を頻繁にコミットする** - 大きな変更を一度にしない
3. **コミットメッセージを分かりやすく書く** - 何を変更したか明確に
4. **同じファイルを同時に編集しない** - 事前に相談する
